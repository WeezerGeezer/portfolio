    Rectangle: Window Management Made Simple

    I've been on MacOS since I purchased a used 2014 Model Macbook Pro during my senior year of high school. It still gets daily use as a music/streaming device for my stereo system, but it served me well for almost 7 years of near daily use. Before that point, I only ever touched Windows PCs and had pretty good experiences with them that made me hesitant to switch to Apple. When I made the switch to MacOS I found the one feature that was missing was the ability to assign applications to certain halves of the screen. Plenty have heaped praises onto this app, and I'll do the same. Rectangle makes it incredibly easy to organize your windows. I started off by usingthe "drag to the side and have the window fill that half of the screen" feature, but I have since become addicted to using their built in shortcuts for managing my windows. It just makes things so quick and snappy, and having the shortcuts memorized (which becomes second nature after a couple minutes of play) makes you look like a total whiz. Pair this with a few terminal windows and you'll have anyone thinking you're some kind of whiz kid.I'll break down the shortcuts I use the most: Pressing "Ctrl-Option" followed by an arrow key reorganizes your window to that side of the current display, and "Ctrl-Option" followed by either U, I, J, or K (each representing a quadrant of the screen) allows you to quickly organize multiple windows into a display at once. The final command I use when I have multiple displays, which is "Ctrl-Option-Command" followed by an arrow to the left or to the right. This lets you move windows across multiple displays, and I've found to be particularly useful when running a double monitor setup.

    For web development specifically, using the shortcut keys to cycle through page widths and making sure that the layouts render corrently in each width has been super useful. There are additional settings in the app that let you change what happens when a command is repeated, I have it to cycle between 1/2, 2/3, and 1/3 displays.

    I like to have most functions available to me from the keyboard, as navigating with a mouse often slows me down. Rectangle enable me to organize my displays with utmost efficiency. Dragging windows to the side is LAME! Take a couple minutes to memorize the fairly intuitive shortcuts and manage your windows like the computer whiz you are. I know I feel like one when I hit a Ctrl-Option-LeftArrow to move a window to the left half of my screenðŸ¤“.

    Firefox Developer Tools: More Than Just Another Browser

    When using MacOS, I use the stock Command-Tab to switch between windows. Given this work flow, using Command-Tab to cycle between apps make it useful to have two web browswers up and running, especially during a web development process. I use Firefox as a standard display for any website I develop. It has built in Dev Tools like Responsive Design Mode, visiaully css adjustments, console for tracking issues output via logging, and network monitor to identify loading bottlenecks. I'll use Safari as well, as most of my shortcuts in Raycast launch into Safari for searches. Having safari as my default "browsing" browser and Firefox as my "developing" browser has made my workflow much more effecient.

    Ditching AltTab: Mastering Native MacOS Window and Tab Switching

    Go to any list of "Useful Mac Apps" and you'll probably find AltTab, a Windows style window switching application that runs at all times. I rejected the need for this, not because I'm paranoid about an app watching my screen (Raycast and Bartender already do this) but because there's a philosophy present in the Mac Spaces Feature that I've found useful for focussing my work and keeping me on track. I organize my windows into different desktop views (Spaces), and use Ctrl-Arrow to switch between spaces. I have one space for writing & code (Usually a Terminal with Vim/Claude Code/Bash on one side, Firefox on the right), one space for Photos, one with Apple Music & Apple Podcasts (I'm a believer in using the stock MacOS apps for the simple stuff, they work well!), and one space that functions as my junk drawer. There's plenty of native macOS shortcuts you can use within these views. For any app that has Tabs, you can swap between them by holding Command and pressing a number between 1 and 9 (Command-N, where N is a number 1-9). This is useful for Terminal, Firefox, and Safari, as long as you keep your number of tabs open to a minimum (I'll do 5 at max to stay focused). Additionally, cycling through app windows using Command-` is great. What's important for me when I'm working is having these workspaces organized, which may mean multiple windows of many apps, pairing this with Rectangles ability to sort windows into quadratns fixes the problem with tab switching and having other windows come to the front.

    A typical portfolio development session demonstrates this workflow perfectly: I'd have Terminal in tab 1 running the local development server (http.server on port 8080), tab 2 ready for git commands, and tab 3 for Claude Code sessions. Command-1, Command-2, Command-3â€”instant switching without thinking. Firefox would display the live site at localhost:8080, and I'd use Command-Tab to jump between Terminal and Firefox dozens of times per session. When I needed to research a CSS technique or check browser compatibility, Command-Tab would bring up Safari (my research browser), I'd find what I needed, then Command-Tab back to Firefox to implement it. The whole danceâ€”Terminal for server status, Firefox for development preview, Safari for researchâ€”became muscle memory. No mouse needed, no visual hunting for the right window, just rapid-fire keyboard shortcuts that kept my hands on the home row and my focus on the work.

    Putting It All Together: The Complete Workflow

    Essentially once I'm sat down to work at my laptop, I'll use a combination of Raycast, Rectangle. Firefox, Claude Code, and my terminal in order to accomplish whatever the goal is I have. I use the Mac Spaces feature to organize my work between editing photos, developing code, and having music or something else on in the background. These tools help me stay focused as I don't have a bunch of clutter clogging up my flow. I'll begin by organizing my windows how I'd need them, and then I'd get to work on whatever the task at hand. Before writing this post, my most recent big project was an update to my personal website, so I chose to outline how these tools have helped me in this specific process.

From initial concept to deployment, the entire site took roughly 15 hours of focused development time spread across a couple weeks. Without these tools? I'd conservatively estimate 30-40 hours, maybe more. The time savings aren't just about raw speedâ€”it's about maintaining momentum and flow.     The best development workflow is one that becomes invisible. You shouldn't be thinking about your tools, they should just WORK. Configure your tools to match your mental model. Experiment until context-switching feels effortless. The hours you spend setting up your environment will pay dividends for years.

    Key Takeaways

    After eight years of refining my MacOS workflow, here's what I've learned works:

    Must-Have Tools:

    • Rectangle - Window management is non-negotiable. Start here.
    • Raycast - Replaces Spotlight and does 100x more. So fun to use and explore.
    • Native Shortcuts - Command-Tab, Command-`, Command-N for tabs. Learn these before installing anything else.

    Nice-to-Have (But Really Nice):

    • Firefox Developer Tools - If you do web development, essential. Otherwise, Safari works fine.
    • Claude Code - Transforms development speed, but requires a learning curve and thoughtful prompting.
    • Spaces - Organizational superpower for focus, though not for everyone.

    For Beginners:

    Start small. Pick one tool, use it for a week until it's muscle memory, then add another. Don't install fifteen productivity apps at once and get overwhelmed. Resist that temptation. Master Rectangle's four basic shortcuts first. Then move to Raycast for launching apps. Build gradually.

    What's Next:

    I'm exploring deeper automation with custom scripts and shell aliases, investigating better clipboard management workflows, and experimenting with Vim motions across all my apps. The workflow never stops evolvingâ€”it's a continuous process of identifying friction and smoothing it out. The goal remains the same: make the tools disappear so the work can shine.

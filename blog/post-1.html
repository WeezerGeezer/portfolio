    Building a Modern Portfolio Site

    A deep dive into creating a responsive, interactive portfolio with modern web technologies

    The Challenge

    Creating a portfolio site in 2024 comes with unique challenges. Users expect fast load times, beautiful animations, and seamless experiences across all devices. Traditional template-based approaches often fall short, leading to generic-looking sites that don't stand out.

    I wanted to build something different—a portfolio that showcased both my technical skills and design sensibility while remaining maintainable and performant.

    Design Philosophy

    I embraced a bento box layout—a design pattern popularized by modern applications. This grid-based approach allows for flexible content organization while maintaining visual hierarchy. Each "box" in the layout serves a specific purpose, from showcasing recent projects to highlighting technical skills.

    The color palette draws from nature: earthy greens (#656D4A) paired with clean whites and grays. This creates a professional yet approachable aesthetic that doesn't overwhelm visitors.

    Technical Implementation

    Rather than reaching for heavy frameworks, I built the site with vanilla HTML, CSS, and JavaScript. This decision paid dividends in performance—the entire site loads in under 2 seconds, even on slow connections.

    Key technical highlights include:

    • CSS Grid and Flexbox: The bento layout uses CSS Grid for the overall structure, with Flexbox handling internal component layout.
    • Progressive Enhancement: The site works perfectly without JavaScript, but enhanced interactions activate when available.
    • WebP Images: All images use modern WebP format with JPEG fallbacks for older browsers.
    • Custom Filtering System: The projects page features a real-time filtering system built from scratch, no libraries needed.

    Responsive Design

    Mobile-first design is non-negotiable in 2024. The site transforms dramatically at different breakpoints, switching from a complex grid layout on desktop to a clean, stacked layout on mobile. The bento boxes flatten into full-width cards, making content scannable on any device.

    Special attention went into the hamburger menu and touch interactions. Every button is adequately sized for thumb navigation, and hover effects gracefully degrade on touch devices.

    Performance Optimization

    Performance wasn't an afterthought—it was baked into every decision. The CSS uses hardware-accelerated transforms for animations, ensuring smooth 60fps interactions. Images are lazy-loaded using the Intersection Observer API, reducing initial page weight.

    The result? A Lighthouse score of 98+ on all metrics. Fast, accessible, and delightful to use.

    Lessons Learned

    Building this portfolio reinforced several principles:

    • Start with content, not design. Know what you want to say before deciding how to say it.
    • Constraints breed creativity. Limiting myself to vanilla JavaScript forced elegant solutions.
    • Details matter. Small touches like smooth transitions and thoughtful spacing elevate the entire experience.
    • Test relentlessly. What works on your laptop might break on someone's phone.

    What's Next

    The site continues to evolve. Future plans include adding a full blog with RSS support, implementing view transitions for smoother page navigation, and exploring more interactive timeline features. The beauty of owning your codebase is the freedom to experiment and iterate.

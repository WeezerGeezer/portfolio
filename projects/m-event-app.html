    M-Event Mobile App

    University Event Management Platform

    Built with React Native & PostgreSQL

    Watch Demo View Slides

    Project Overview

    A comprehensive mobile application designed for University of Michigan students to discover, create, and manage campus events with real-time updates and social features. This full-stack project demonstrates proficiency in mobile development, database design, and collaborative software engineering.

    The Challenge

    University students needed a centralized platform to discover and manage campus events, but existing solutions were fragmented and lacked real-time coordination:

    • Event Discovery: Students struggled to find relevant campus events across multiple channels and platforms
    • Real-Time Updates: Need for instant notifications about event changes, cancellations, and new opportunities
    • Scale & Performance: Supporting thousands of concurrent users during peak times like orientation and major events
    • Data Consistency: Maintaining accurate event information across mobile clients with offline capabilities
    • User Experience: Creating intuitive interfaces for both event organizers and attendees

    Technical Solution

    Cross-Platform Mobile App

    React Native application providing native performance on both iOS and Android with shared codebase for efficient development.

    Robust Backend Architecture

    Node.js and Express server with RESTful API design, supporting high-concurrency user interactions and data consistency.

    Database Design

    PostgreSQL database with optimized schema for complex event relationships, user management, and real-time query performance.

    Real-Time Communication

    WebSocket implementation for instant notifications, event updates, and live attendee tracking across all connected devices.

    Technical Challenges Overcome

    Database Performance

    Challenge: Complex queries across multiple tables were causing slow response times during peak usage.

    Solution: Implemented database indexing strategies, query optimization, and connection pooling to achieve sub-200ms response times.

    Real-Time Synchronization

    Challenge: Keeping event data consistent across thousands of mobile clients with varying network conditions.

    Solution: Developed WebSocket-based real-time updates with offline-first architecture and conflict resolution.

    Cross-Platform Consistency

    Challenge: Maintaining identical user experience between iOS and Android while leveraging native capabilities.

    Solution: Created shared component library with platform-specific optimizations and consistent design system.
